# Incident Response Playbook

## Overview

This playbook provides a structured approach for investigating alerts generated by the Sysmon Threat Detection & Process Monitoring Dashboard. It helps SOC analysts perform consistent investigations using endpoint telemetry collected through Sysmon and analyzed in Splunk Enterprise.

---

# Incident Response Lifecycle

1. Detection
2. Validation
3. Investigation
4. Containment
5. Eradication
6. Recovery
7. Lessons Learned

---

# Detection Scenarios

The dashboard currently monitors:

- PowerShell Execution
- Command Prompt Execution
- Suspicious Process Execution
- High Integrity Processes
- Network Connections
- Registry Modifications

---

# Playbook 1 – PowerShell Execution

## Trigger

PowerShell process detected.

### Investigation

- Identify the user.
- Review the parent process.
- Inspect the full command line.
- Check integrity level.
- Review network connections.
- Search for additional PowerShell executions.

### Containment

- Isolate the endpoint if malicious.
- Disable compromised accounts if necessary.

### Recovery

- Remove malicious scripts.
- Validate endpoint integrity.
- Continue monitoring.

---

# Playbook 2 – Suspicious Process Detection

## Trigger

Execution of:

- powershell.exe
- cmd.exe
- rundll32.exe
- regsvr32.exe
- certutil.exe
- mshta.exe
- wmic.exe

### Investigation

- Verify process legitimacy.
- Review parent-child process relationship.
- Inspect command-line arguments.
- Review registry activity.
- Check outbound network connections.

### Containment

- Terminate malicious process.
- Block malicious hash if available.
- Isolate affected endpoint.

---

# Playbook 3 – Registry Modification

## Trigger

Registry value modification detected.

### Investigation

- Review modified registry path.
- Identify responsible process.
- Check if persistence keys were modified.
- Review previous registry activity.

### Containment

- Restore registry value.
- Remove unauthorized persistence.
- Continue monitoring.

---

# Playbook 4 – High Integrity Process

## Trigger

Process executed with High or System integrity.

### Investigation

- Confirm administrative activity.
- Verify user account.
- Review parent process.
- Check for privilege escalation.

### Containment

- Disable compromised account if required.
- Investigate additional elevated processes.

---

# Playbook 5 – Network Connection

## Trigger

Outbound connection detected.

### Investigation

- Review destination IP.
- Verify destination hostname.
- Identify process creating the connection.
- Correlate with process creation events.

### Containment

- Block malicious destination.
- Isolate endpoint if required.
- Continue monitoring.

---

# Evidence Collection

During every investigation collect:

- Timestamp
- Username
- Hostname
- Process Name
- Parent Process
- Command Line
- Integrity Level
- Destination IP
- Registry Path
- Related Sysmon Event IDs

---

# Analyst Checklist

- Validate the alert.
- Review process lineage.
- Correlate related events.
- Identify affected users.
- Determine attack scope.
- Document findings.
- Escalate if malicious activity is confirmed.

---

# Outcome

This playbook provides a repeatable investigation workflow that enables SOC analysts to respond consistently to alerts generated by the Sysmon Threat Detection Dashboard.
